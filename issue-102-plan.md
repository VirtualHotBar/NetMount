# NetMount Issue #102 改造计划

关联 Issue: https://github.com/VirtualHotBar/NetMount/issues/102

## 1. 背景与目标

Issue #102 的核心诉求是把挂载能力从“能挂”升级到“可调优”：

- macOS 挂载后端应有清晰选择与推荐（优先 `nfsmount`，其次 `fuse-t`）。
- `--vfs-cache-mode` 需要给出明确策略（`off/minimal/writes/full`）与场景说明。
- 面向高级用户，保留足够可控参数；面向普通用户，提供稳定默认值和低认知成本入口。

本计划目标：

1. 统一当前参数默认值与 UI 语义，消除冲突。
2. 增加平台化配置能力（重点是 macOS 后端策略）。
3. 提供文档和诊断支持，降低误配置与排障成本。

## 2. 现状问题

当前代码中存在以下不一致：

- `CacheMode` 默认值冲突：
  - `src/controller/storage/mount/parameters/defaults.ts` 中 `defaultVfsConfig.CacheMode = 'writes'`
  - 同文件 `vfsCacheModeParam.default = 'minimal'`
  - `src/page/mount/add.tsx` 高级表单 `overwriteValues` 里又把 `CacheMode` 固定成 `full`
- 结果是用户看到的默认项与最终保存值可能不一致。
- macOS 相关挂载后端未形成明确的可配置策略入口（issue 关注点未落地）。

## 3. 里程碑

## P0：默认值与语义统一（优先级最高）

### 范围

- 仅处理现有参数体系，不新增复杂字段。
- 目标是“默认值一致、行为可预测”。

### 改动项

1. 统一 `CacheMode` 默认值来源，移除多处互相覆盖。
2. 基础模式默认策略设为 `writes`（兼顾可用性与风险）。
3. 高级模式中 `CacheMode` 不再被硬编码覆盖。
4. 根据 `CacheMode` 动态显示/强调关键缓存参数（如 `CacheMaxSize`、`CacheMaxAge`）。

### 验收标准

- 新增挂载时，UI 默认值与保存到配置的值一致。
- 进入编辑页再保存，不应发生 `CacheMode` 被静默改写。
- 不改其他参数时，回归测试可保持现有挂载成功率。

---

## P1：平台化挂载策略（对应 issue #102 核心）

### 范围

- 增加 macOS 挂载后端选择能力（仅在 macOS 显示）。
- 补齐策略型预设，降低手工调参难度。

### 改动项

1. 扩展挂载参数模型（`mountOpt`）：
   - 增加用于后端选择的字段（例如 `MountType`）。
2. macOS 默认策略：
   - 默认 `nfsmount`。
   - 不可用时回退 `fuse-t`（回退策略可在后续增强为运行时探测）。
3. 新增“场景预设”：
   - 性能优先
   - 兼容优先
   - 低占用优先
4. 预设覆盖最小必要参数：
   - `CacheMode`
   - `CacheMaxSize`
   - `WriteBack`（及必要联动参数）

### 验收标准

- macOS 用户在新增/编辑挂载时可见后端选择项。
- 默认值符合策略（优先 `nfsmount`）。
- 应用预设后，参数联动正确且可被用户继续手动修改。

---

## P2：文档与可观测性补齐

### 范围

- 以“降低踩坑”为目标，不引入复杂新功能。

### 改动项

1. 在挂载页面增加关键提示：
   - `full` 模式的磁盘占用风险
   - 不同模式的适用场景
2. README/文档新增“macOS 后端选择 + CacheMode 指南”。
3. 诊断信息中增加挂载关键参数快照（便于远程排障）。

### 验收标准

- 用户可在 UI 或文档中快速找到模式选择建议。
- 反馈问题时可附带关键参数，开发侧能复现大部分配置类问题。

## 4. 推荐实施顺序

1. 先做 P0（1 个小版本内完成）。
2. 再做 P1（1 个小版本内完成，必要时拆成两次 PR）。
3. 最后做 P2（可与 P1 并行推进文档部分）。

## 5. 风险与回滚

### 风险

- 默认值调整可能改变部分老用户行为预期。
- macOS 后端策略若处理不当，可能引入平台兼容问题。

### 缓解

- 对已有配置做“非破坏性升级”：仅在新建挂载使用新默认值，旧配置保持原值。
- 关键策略字段保留手工覆盖能力。
- 在 PR 中补充跨平台回归清单（Windows/macOS/Linux）。

### 回滚策略

- 若出现大面积兼容问题，可先回滚策略默认值，不回滚字段结构。
- 保留高级选项入口，确保用户可手动绕过策略层。

## 6. 拆分到 PR 的建议

1. PR-1（P0）：默认值统一 + 表单覆盖修复 + 基础回归测试
2. PR-2（P1）：macOS 后端字段 + 平台 UI + 预设策略
3. PR-3（P2）：文档、提示文案、诊断增强
